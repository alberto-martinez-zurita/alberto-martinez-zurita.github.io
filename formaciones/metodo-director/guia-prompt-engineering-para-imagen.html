Fundamentos Visuales

Para controlar la generación de imágenes, es crucial dominar tanto los conceptos de los modelos de IA como el lenguaje técnico de la fotografía y el cine. Esta sección explora la relación entre texto e imagen y proporciona un glosario interactivo de los términos que otorgan un control preciso sobre el resultado visual.

Modelos y Backends

Conocer las fortalezas de cada modelo es clave. Algunos, como Midjourney, son excelentes para estilos artísticos, mientras que otros como SDXL o Imagen ofrecen un control más granular para fotorrealismo y composición.

Semántica Latente

Los modelos no "entienden" el texto como los humanos. Traducen palabras a un espacio de conceptos latente. La ambigüedad en el texto (ej. "banco") genera incertidumbre en la imagen. La precisión es fundamental.

Condicionamiento

Cada palabra en el prompt "condiciona" o guía la generación. Los términos más importantes deben ir al principio. Ténicas como el weighting permiten dar más o menos fuerza a cada concepto.

Anatomía de un Prompt Visual Efectivo

Un prompt bien estructurado es predecible y controlable. Sigue una jerarquía lógica que va de lo general a lo específico, del sujeto principal a los detalles técnicos de la "cámara virtual". Usar esta estructura reduce la ambigüedad y aumenta la fidelidad al concepto deseado.

Jerarquía del Prompt

1. Sujeto
→
2. Atributos
→
3. Entorno
→
4. Iluminación
→
5. Estilo/Medio
→
6. Cámara/Lente
→
7. Parámetros

Ejemplo: Un astronauta con un traje reflectante, flotando en una nebulosa cósmica, iluminado por una estrella de neutrones azul, pintura al óleo digital, lente gran angular de 35mm.

Bueno vs. Ambiguo

La especificidad es la clave para obtener resultados predecibles.

✗ Ambiguo:

"Un hombre en un banco."

✓ Bueno:

"Un anciano sentado en un banco de madera de un parque, alimentando palomas, fotografía callejera, lente de 50mm."

Prompts Negativos

Utiliza prompts negativos (`--no` o `negative_prompt`) para eliminar elementos no deseados. Son cruciales para limpiar artefactos, evitar estilos no deseados o corregir errores comunes del modelo (como manos con 6 dedos).

Ejemplo Negativo: --no malformado, mala anatomía, manos feas, texto, marcas de agua, borroso, baja calidad

Técnicas Avanzadas de Control Visual

Más allá de la estructura básica, existen técnicas avanzadas para un control casi total sobre la composición, el estilo y la coherencia de la imagen. Estas herramientas son fundamentales para flujos de trabajo profesionales y la generación de contenido específico.

Permite dar más o menos importancia a ciertas palabras. La sintaxis varía (ej. `(palabra:1.4)` en SD, `palabra::1.4` en Midjourney). Un peso > 1 aumenta su influencia, < 1 la reduce. Los embeddings (como en Textual Inversion) permiten "enseñar" un nuevo concepto o estilo al modelo con unas pocas imágenes de ejemplo, que luego puede ser invocado con una sola palabra clave.

ControlNet es una técnica revolucionaria que condiciona la generación de la imagen a partir de una imagen de entrada (ej. un boceto, un mapa de profundidad, una pose humana). Permite un control compositivo sin precedentes. El flujo i2i estándar usa una imagen inicial y un prompt para generar variaciones, conservando la composición y colores generales. Ajustar el `denoising strength` controla cuánto se aleja de la imagen original.

  • CFG Scale (Classifier-Free Guidance): Controla la adherencia al prompt. Valores bajos (3-6) dan más creatividad al modelo. Valores altos (8-15) fuerzan una interpretación más literal, pero pueden causar sobre-saturación.
  • Steps (Pasos de Muestreo): Número de iteraciones. Más pasos suelen dar más detalle, pero con rendimientos decrecientes. 20-30 es un buen punto de partida para la mayoría de samplers.
  • Sampler: El algoritmo para el proceso de "denoising". `Euler a` es rápido y creativo. `DPM++ 2M Karras` es un estándar de alta calidad. `LCM` es extremadamente rápido para previsualizaciones.

Evaluación y Métricas Visuales

En un entorno de producción, la calidad visual no puede ser subjetiva. Se utilizan métricas cuantitativas para evaluar la coherencia, la estética y la fidelidad de las imágenes generadas. Este proceso es clave para el testing A/B de prompts y para mantener la calidad del sistema a lo largo del tiempo.

CLIP Similarity

Mide la similitud semántica entre el prompt (texto) y la imagen generada. Un score alto indica una buena correspondencia conceptual. Es la métrica principal para validar la fidelidad a las instrucciones.

Aesthetic Score

Un modelo entrenado para predecir qué tan "estéticamente agradable" es una imagen para un humano. Se usa para filtrar automáticamente las generaciones de baja calidad visual o poco atractivas.

FID (Fréchet Inception Distance)

Compara la distribución de las imágenes generadas con un set de datos de imágenes reales. Un FID más bajo significa que las imágenes son más realistas y diversas. Mide la calidad general del modelo.

LPIPS (Learned Perceptual)

Mide la similitud perceptual entre dos imágenes. Es útil en tareas i2i para asegurar que la imagen de salida no se desvía demasiado de la de entrada en términos de estructura y contenido.

Consistencia de Atributos

Verifica la presencia de objetos o características específicas mencionadas en el prompt. Se puede automatizar con modelos de detección de objetos o VLMs (Vision Language Models).

LLM-as-a-Judge

Se utiliza un VLM potente (como GPT-4o o Gemini) para "juzgar" la calidad de una imagen con respecto a una checklist de criterios complejos (ej. "¿La iluminación coincide con 'luz de mañana dorada'?").

Operativa de Producción (LLMOps/VLM Ops)

Integrar la ingeniería de prompts en un ciclo de vida de desarrollo de software requiere herramientas y procesos específicos. La operativa de producción (Ops) se encarga de versionar, testear, desplegar y monitorizar los prompts como si fueran código, asegurando la robustez y escalabilidad del sistema.

CI/CD para Prompts Visuales

Despliegue (Canary)

El nuevo prompt se despliega a un % pequeño de usuarios. Se monitorizan las métricas en tiempo real.

→
④

Rollout / Rollback

Si las métricas son positivas, se despliega al 100%. Si no, se revierte automáticamente (rollback).

Seguridad y Ética

La operativa de prompts debe incluir barreras de seguridad (guardrails) para mitigar sesgos, prevenir la generación de contenido dañino y detectar jailbreaks. Se usan prompts defensivos y clasificadores para filtrar entradas y salidas problemáticas.

Integraciones y Herramientas

Los flujos de trabajo se implementan usando APIs de modelos (OpenAI, Google) o interfaces de código abierto como ComfyUI o Automatic1111, que permiten construir grafos de generación complejos y automatizar los pipelines de testing y generación.
